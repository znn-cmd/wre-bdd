# Уведомления партнёру в Telegram при передаче лида

Кратко: приложение отправляет сообщение в чат Telegram партнёра **когда у лида поле `transfer_status` становится одним из настроенных кодов** (по умолчанию **`sent`** — обычно это статус вроде *Sent to partner* / «передан партнёру» в справочнике Statuses).

---

## Как это устроено в коде

1. **Создание лида** (`createLeadAction`): если при создании сразу указан `transfer_status`, попадающий в список триггеров (в форме сейчас по умолчанию уходит `new` — уведомления **не будет**, пока менеджер не сменит статус).
2. **Обновление лида** (`updateLeadAction`): если **`transfer_status` изменился** и **новое значение** входит в список триггеров (например, с `new` на `sent`), вызывается `notifyPartnerNewLead`.
3. Отправку инициируют роли: **`our_manager`**, **`admin`**, **`rop`**, **`partner_dept_manager`** (действия других ролей не шлют Telegram).

Файлы:

- `server/telegram/notify-new-lead.ts` — текст сообщения, вызов Bot API, запись в лист **Telegram_Log**.
- `server/telegram/notify-rules.ts` — какие коды `transfer_status` считаются «передано партнёру» и какие роли могут триггерить отправку.
- `server/actions/leads.ts` — вызов уведомления после создания/обновления лида при выполнении условий.

---

## Что настроить у себя

### 1. Лист **Statuses** (Google Таблица)

- В категории **`transfer_status`** должен быть статус с **кодом**, по которому вы хотите слать Telegram.
- В типовой конфигурации это код **`sent`** (подпись в UI может быть любой, например *Sent to partner*).
- Если у вас другой **status_code** (не `sent`), задайте переменную окружения (см. ниже).

### 2. Лист **Partners**

Для партнёра, которому уходят лиды:

| Колонка | Назначение |
|--------|------------|
| `telegram_chat_id` | ID чата (или пользователя), куда писать боту. Узнать у [@userinfobot](https://t.me/userinfobot) / [@getidsbot](https://t.me/getidsbot) или для группы — добавить бота в группу и взять id группы (часто отрицательный). |
| `telegram_bot_token` | Опционально: свой токен бота для этого партнёра. Если пусто, используется общий токен из env. |

### 3. Переменные окружения (Vercel / `.env`)

| Переменная | Обязательность | Описание |
|------------|----------------|----------|
| `TELEGRAM_DEFAULT_BOT_TOKEN` | Рекомендуется | Токен бота [@BotFather](https://t.me/BotFather), если у партнёров не заполнен свой `telegram_bot_token`. |
| `APP_BASE_URL` | Рекомендуется | Публичный URL приложения (в сообщении даётся ссылка на карточку лида). На Vercel часто хватает `VERCEL_URL`. |
| `TELEGRAM_NOTIFY_TRANSFER_STATUS_CODES` | Опционально | Через запятую коды `transfer_status`, при **установке** которых уходит письмо. По умолчанию: **`sent`**. Пример: `sent,handed_to_partner`. |

### 4. Сценарий для менеджера WRE

1. Создать лид (страна, партнёр, клиент, телефон, email, язык, комментарий, CRM и т.д.) — `transfer_status` обычно **`new`**.
2. Когда клиент реально **передан партнёру**, в таблице лидов выставить **Transfer** в статус с кодом **`sent`** (или вашим кодом из `TELEGRAM_NOTIFY_TRANSFER_STATUS_CODES`).
3. После сохранения партнёр получит сообщение в Telegram (если настроены `chat_id` и токен).

---

## Логи и отладка

- Лист **`Telegram_Log`**: каждая попытка — строка со статусом `sent`, `failed`, `skipped` и текстом ошибки при сбое.
- **`skipped`**: нет токена бота или пустой `telegram_chat_id` у партнёра (и нет дефолтного токена в env).
- Текст ошибки Telegram API попадает в `error_message`.

---

## Важно

- Уведомление привязано к **коду** в колонке лида `transfer_status`, а не к русской подписи в интерфейсе. Подпись берётся из справочника, код должен совпасть с настройкой (`sent` или ваш список в env).
- Один переход «старый код → триггер-код» даёт **одну** отправку при сохранении; повторное сохранение без смены `transfer_status` не дублирует сообщение.
- Пароли и чувствительные поля в Telegram не отправляются; в сообщении есть телефон/email клиента — учитывайте политику конфиденциальности.
