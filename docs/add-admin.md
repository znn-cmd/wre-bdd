# Как добавить администратора

Роль задаётся в листе **Users**, колонка **`role`** — значение должно быть **точно** `admin` (латиница, нижний регистр). Вход в приложение — только по **magic link**: в таблице хранится **SHA-256** токена, сам токен показывается один раз при создании пользователя в UI или задаётся вручную при «первом» админе.

## Что нужно заранее

- Выполнены `npm run init-sheet` и заполнены минимальные справочники (см. `docs/seed-demo.md`).
- В `.env.local` заданы `GOOGLE_SHEETS_SPREADSHEET_ID`, ключ сервисного аккаунта (`GOOGLE_SERVICE_ACCOUNT_JSON_FILE` или `GOOGLE_SERVICE_ACCOUNT_JSON`), **`SESSION_JWT_SECRET`** (не короче 32 символов).
- Таблица расшарена на email сервисного аккаунта с правом **Редактор**.

## Способ 1. Первый админ (вручную в Google Таблице)

Пока нет ни одного пользователя с рабочей ссылкой, админа добавляют **строкой на листе Users**.

1. Откройте лист **Users** (заголовки уже из `init-sheet`).
2. Добавьте строку. Обязательно заполните как минимум:

   | Колонка | Значение |
   |---------|----------|
   | `user_id` | любой уникальный id, например `U-ADMIN-1` |
   | `full_name` | отображаемое имя |
   | `role` | `admin` |
   | `is_active` | `true` |
   | `token_hash` | **ровно 64 символа** `0-9` и `a-f` (hex SHA-256 UTF-8 токена). Не путать с Base64 и случайными буквами вроде `X`, `Y`, `Z`, `K` — в hex их нет. |
   | `token_expires_at` | лучше оставить **пустым** (без срока) или формат **ISO**: `2027-03-30`. Формат `30.03.2027` приложение **не понимает** (дата считается недействительной и срок не применяется). |
   | `created_at`, `updated_at` | дата/время в ISO, например `2026-03-31T12:00:00.000Z` |

3. Остальные колонки для админа обычно **пустые**: `partner_id`, `source_manager_id`, `allowed_country_codes`, `allowed_partner_ids`, `token_expires_at` (или пусто = без срока), `telegram_chat_id_optional`, `dashboard_preset_json` можно оставить пустыми / `{}` по необходимости.

4. **Токен и хеш:** придумайте длинную случайную строку (токен) — её **никуда в таблицу не пишите**, только хеш.

   **Node.js** (из корня проекта):

   ```bash
   node -e "console.log(require('crypto').createHash('sha256').update('ВАШ_СЕКРЕТНЫЙ_ТОКЕН','utf8').digest('hex'))"
   ```

   Скопируйте вывод в колонку **`token_hash`**.

5. Запустите приложение: `npm run dev`.
6. Откройте ссылку входа.

   **Критично:** в адресе `/access/...` должен быть **исходный секретный токен** (та строка, от которой вы считали SHA-256). **Нельзя** подставлять значение колонки **`token_hash`** из таблицы — это уже хеш; от него считается другой SHA-256, пользователь не находится. Если в URL 64 символа `0-9a-f`, это почти наверняка ошибка.

   **Важно:** в токене нельзя без кодирования использовать символы **`&` `#` `%` `?`** и т.п. — иначе браузер сломает URL (часто **400 Bad Request**), как с `6Z59&KDT5...`.

   - **Проще всего:** токен только из **букв, цифр, `-` `_`** (без пробелов). Тогда достаточно:
     `http://localhost:3000/access/<токен>`
   - Если токен уже с «опасными» символами — **закодируйте целиком** и подставьте в путь:
     ```bash
     node -e "console.log(encodeURIComponent('ВАШ_ТОКЕН'))"
     ```
     Откройте: `http://localhost:3000/access/<вывод_команды>`
   - Либо один раз откройте (токен в query уже в закодированном виде):
     `http://localhost:3000/app/dashboard?token=<encodeURIComponent(токен)>`
     Middleware перенаправит на `/access/...` с правильным кодированием.

   Хеш в таблице считается от **исходного** текста токена (до кодирования в URL).

После успешного входа откроется дашборд; пункт **Пользователи** (`/app/users`) доступен только роли **admin**.

## Способ 2. Следующие админы (уже есть админ)

1. Войдите под существующим **admin** по ссылке `/access/...`.
2. Откройте **Пользователи** в меню приложения.
3. Создайте пользователя, в поле роли выберите **`admin`**.
4. Интерфейс **один раз** покажет ссылку доступа (plain token) — сохраните её и передайте новому админу безопасным каналом. В таблице остаётся только хеш.

Смена роли существующего пользователя на `admin` — там же, через редактирование пользователя.

---

## Важно

- Значение роли в таблице: **`admin`**, не «Админ» и не `Admin`.
- Только **admin** может управлять пользователями (`manage_users`) и критичными настройками; роль **`rop`** шире по лидам, но **не** открывает админку пользователей.
- При утечке ссылки зайдите под админом в **Пользователи** и выполните **ротацию токена** — старая ссылка перестанет работать.

Подробнее про права: `docs/rbac.md`.
